## Milestone Vocabulary

| Milestone | Meaning |
|---|---|
| Concept Gate | Product pillars, audience, period, and non-goals are approved |
| Pre-production Exit | Technical risks are reduced and pipelines can support production |
| First Playable | The smallest end-to-end interactive loop works |
| Vertical Slice | A representative, polished regional sample proves final quality direction |
| Alpha | All 1.0 systems are implemented and connected |
| Content Complete | Planned 1.0 content is present and data validation passes |
| Beta | Feature work is locked; focus moves to bugs, balance, UX, and performance |
| Release Candidate | A build can ship unless a release-blocking issue is found |

## Phase Status Rules

Use one of these states:

- Not started.
- Discovery.
- In production.
- Validation.
- Complete.
- Blocked.
- Deferred.

Only one major gameplay phase should be the critical-path production focus at a time. Supporting workstreams may operate in parallel when their dependencies are clear.
